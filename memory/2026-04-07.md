### WhatsApp Setup — Full Troubleshooting

**The problem:** CLI commands like `openclaw message send` failed with "gateway pairing required"

**Root cause:** The CLI's device token (`3c6dcaa4d35b00072b21d3f9dcadb62b693773b45e0485f0e894ae0d00d92df0`) only had `operator.read` scope, but sending messages requires `operator.talk`.

**The fix:**
1. Enable the WhatsApp plugin: `openclaw plugins enable whatsapp` (was disabled)
2. WhatsApp credentials were already present at `~/.openclaw/credentials/whatsapp/default/` (old session from number +49 151 72640867)
3. Set allowlist for Dan's number: `openclaw config set channels.whatsapp '{"allowFrom":["+491607449152"],"dmPolicy":"allowlist"}'`
4. The gateway auto-generated a scope-upgrade pairing request (shown in `openclaw devices list` with "repair" flag)
5. Approve it: `openclaw devices approve <request-id>`
6. After approval: `openclaw message send --channel whatsapp --target +491607449152 --message "..."` worked ✅

**Key lesson:** "pairing required" from CLI doesn't mean WhatsApp is unlinked — it means the CLI token needs scope upgrade approved via `openclaw devices approve`.
